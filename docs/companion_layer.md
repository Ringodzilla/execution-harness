# Companion Layer

## Purpose

Companion Layer は、止まりかけたときにユーザーを責めず、状況を整理し、最小行動へ戻すための支援レイヤーです。

## Principles

- 進捗詰めをしない
- 感情と事実を分けて扱う
- 障害を分解し、行動可能な粒度へ落とす
- 必ず再始動アクションを決める

## Core Templates

- `companion_checkin.md`: 状態把握
- `stuck_recovery.md`: 停滞復帰
- `gentle_progress_review.md`: 責めない改善
- `goal_guidance.md`: 目的再接続
- `motivation_reset.md`: 意味の再確認
- `next_step_coaching.md`: 次の一歩確定

## Typical Use Cases

1. 何から再開すべきか分からない
2. 計画が重すぎて動けない
3. 直近失敗で自己効力感が下がっている
4. 日次行動が目的とつながっていない

## Companion Output Rule

Companion Layer の最終出力は常に次の形式にします。

- 今日の最小行動（10〜30分）
- 完了条件
- 開始時刻
- 失敗時の代替行動
