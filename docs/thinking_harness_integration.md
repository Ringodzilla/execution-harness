# thinking-harness Integration

## Input Contract

thinking-harness から受け取る最小入力:

- strategy: 目指す方向性
- center_pin: 最優先で守る軸
- hypothesis: 成果に効く仮説群

## Translation Rule

execution-harness では次の順で変換します。

1. strategy → 年間テーマ/North Star
2. center_pin → やらないこと/優先順位ルール
3. hypothesis → KPI/四半期施策/週次検証タスク

## Feedback Contract

execution-harness から thinking-harness へ返す情報:

- 実行結果（達成/未達）
- KPI推移
- KPTのProblem/Try
- YWTの学習ポイント

この往復で「考える」と「実行する」を分断せず、継続的に強化します。

## Restart Decision

以下が続く場合は thinking-harness へ再接続して再設計します。

- 仮説の再現性が低い
- KPI改善が見られない
- center_pin と実行が矛盾している
