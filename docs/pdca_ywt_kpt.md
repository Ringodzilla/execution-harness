# PDCA / YWT / KPT

## PDCA in execution-harness

- Plan: 年間〜週次で目的と指標を設計
- Do: 日次で最重要行動を実行
- Check: KPI と実行率、障害を確認
- Act: 翌週・翌月へ改善反映

## KPT (System Improvement)

KPT は、努力量の評価ではなく運用システムの改善に使います。

- Keep: 再現すべき機能した行動
- Problem: 前進を阻害した仕組み上の課題
- Try: 次サイクルで試す改善

## YWT (Learning Language)

YWT は、経験を学習資産へ変えるための言語化です。

- やったこと: 事実
- わかったこと: 解釈・知見
- 次やること: 行動変換

## Practical Loop

1. 週末に KPT/YWT を実施
2. Try を次週計画へ転記
3. 学習ポイントを翌月方針へ集約
4. KPI傾向と照らして優先順位を再設定
5. 「次の一歩」を翌営業日最初の5分行動として固定
